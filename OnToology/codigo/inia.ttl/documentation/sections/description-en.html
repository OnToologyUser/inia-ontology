The main objective of the ontology is to represent the metadata of the dataset stored in the open data portal of INIA, taking into account the metadata characteristic of this type of data sets. For example, metadata may include information about the taxonomic area over which data is collected in the data set or the software with which the data should be processed. However, a large number of metadata of these data sets are common to other existing catalogs and therefore ontologies that cover such metadata already exist. After a search it was observed that the dataid ontology is currently the most complete to represent metadata of catalogs and datasets while reusing domain ontologies for specific topics such as languages, licenses, etc. To complete the rdf representation of the necessary metadata, other ontologies such as Organization ontology or FOAF have been reused.
